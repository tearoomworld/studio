Source · Audit

v1 → v2 compatibility audit.

Risks specific to Source's marketplace mechanics. Multi-city schema, Stripe Connect, App Store IAP boundaries, Capacitor wrap — all locked in week 1 so v2 (attendee accounts, event series, multi-city) ships as an additive release.

Contract
The eight Source compatibility rules.
Follow these during v1 and v2 (attendee accounts, event series, multi-city) ships as an additive release with no migration and no breaking native-app changes.

  1. Every event row has a city from day one. v1 writes 'atlanta'; v2 unlocks the column.
  2. Every event row has a nullable series_id from day one. v1 always null. v2 lights up event series.
  3. users.user_type allows 'attendee' from day one. v1 only creates 'host' + 'founder'.
  4. Tickets store buyer_email + nullable user_id. v2 stitches anonymous purchases to accounts retroactively.
  5. Every UI surface checks a feature flag. v2 surfaces ship dark behind city.flags.attendee_accounts.
  6. Stripe Connect from day one, not later. Can't add Connect to a marketplace after launch. Approval takes 3-7 days.
  7. Native shell stays thin. Capacitor wraps the web app. Native code only for: push, share, deep links.
  8. App Store IAP boundary clearly drawn. Physical event tickets exempt; digital subscriptions are not.
High
Multi-city must exist in schema from day one.
Today: Source is Atlanta-only. Tempting to hard-code "atlanta" in the UI.

Why it breaks v2: The moment you list your first Austin event, every query that filters events needs to learn about cities. Doing this with live money and active hosts means downtime hosts can't tolerate.
v1 schema: every event row has city text not null default 'atlanta'. URLs include /atl/event/[slug] from day one. When Nashville opens, you add an enum value, not a migration.
High
Stripe Connect can't be retrofitted to a marketplace.
Why it breaks v1: Regular Stripe routes payment to you and you owe hosts manually. Connect routes payment splits automatically at purchase. Switching mid-stream means refunding every ticket and re-onboarding every host.
Week 1: form LLC, then submit Stripe Connect application immediately. By week 3 (checkout build), Connect is approved.
High
App Store IAP rules are existential if misread.
Apple's policy: "Real-world experience" tickets are exempt from IAP. Digital subscriptions and content are not.

Why it's load-bearing: v2 features like "follow a host for $4.99/month" or "premium attendee accounts" are digital subscriptions. Those must use Apple's IAP (30% cut, 15% after year one). A single misclassified product flow can get the entire app pulled.
v1 ships with Stripe-only checkout for tickets to physical events. Zero digital subscriptions in v1. v2 designs any digital product knowing it has to live behind IAP — separate code paths, separate revenue accounting.
Medium
Capacitor wrap is fine — until v2 needs deep native.
Why it might matter: If v2 wants background notifications, complex camera, AR previews, or offline mode, Capacitor strains and you might need React Native.
v1 architect: use platform-agnostic data fetching, avoid CSS that doesn't translate to RN styles, keep component tree shallow. v2 question gets answered in month 3.
Medium
Native app updates ship on Apple's schedule, not yours.
Why it matters before v2: Any bug fix touching the native shell needs app store review (1-7 days). Web app code can hot-fix via Vercel; native bits cannot.
Native shell does the minimum (push, share, deep links). Everything else lives in web. CI check that flags new native code.

Source · Agents

Agents + Claude Skills.

Where AI does the real work in Source. Host-facing agents (listing polisher, onboarding, pricing). Founder-facing agents (content generator, host scouting). Claude Skills that encode the editorial aesthetic. Safety rails for marketplace integrity.

Event description polisher

The #1 quality lever on the marketplace. Host writes 1-2 sentences; agent drafts a richer listing in Source's voice. Host reviews + edits before save.

Build an event description polisher at apps/hosts/app/api/polish-listing/route.ts. Input: {raw_title, raw_description, event_type}. Output: {polished_title, polished_description, suggested_tags}. Claude API with Source's voice rules: lowercase, editorial, no marketing speak. Preserve all factual details. Show in "review + edit" UI before saving.

Host onboarding agent

Walks new hosts through Stripe Connect setup, first event creation, ticket pricing. Conversational > 5-step forms.

Ticket pricing suggester

Based on similar atlanta events, suggests a price band. Host always overrides.

Post-event recap email

24h after event ends, agent drafts a 2-sentence recap from host's notes + photos. Asks for rating. Never fabricates moments.

Content idea generator

Based on this week's atlanta events: 3 IG caption ideas + 2 TikTok hooks. You edit and ship. The agent doesn't post — you do.

Build a content idea generator at apps/admin/app/api/content-ideas/route.ts. Weekly cron, Monday morning. Fetch events from past 7 days + next 7 days. Claude API drafts 3 IG captions + 2 TikTok hooks based on actual events. Saved to content_drafts table. Voice rules: lowercase, specific, no marketing speak. Never fabricate. Empty output for empty weeks.

Host scouting agent

Scrapes public atlanta event listings (IG explore, Eventbrite atl, Partiful) and surfaces hosts you don't know about. The agent never sends a message — that's your job, in person or DM.

Weekly atlanta state-of-the-marketplace digest

Monday mornings. What worked this week, hosts at risk, top-rated events. Terse, actionable, no fluff.

Post-event sentiment summarizer

All ratings + comments summarized in one paragraph for the host. Helps them improve. Helps you spot patterns across the marketplace.

source-component

Editorial aesthetic: Helvetica Neue everywhere, all lowercase, sharp corners (max 4px), hairline borders, no shadows. Triggered for any UI work.

Create .claude/skills/source-component/SKILL.md. Font: Helvetica Neue. All interface text lowercase via CSS. Color tokens: --ink #0a0a0a, --bg #ffffff, --soft #f4f4f2, --hair #e8e8e8. Border radii max 4px. No shadows, no gradients. Spacing scale: 4/8/12/16/20/24/32/40. Anti-patterns: emoji, pastel, rounded buttons, drop shadows, marketing copy, exclamation points.

source-voice

Lowercase. No exclamation marks. Concrete over abstract. Banned: "platform", "experiences", "curated", "leverage". Preferred: specific times, places, people.

Pre-flight host listing check

Profanity, scam keywords, mismatched dates. Deterministic, fast. Blocks publish until host fixes.

Automatic refund hold

If host hasn't completed Stripe Connect onboarding when their event happens, ticket payouts are held until they finish. Protects buyers, motivates hosts.

Same as Kindred. Per-endpoint daily cap. Slack alerts at 80%. Solo founder protection.
