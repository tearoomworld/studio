Studio · Audit

v1 → v2 compatibility audit.

Run this before you lock v1 schema or UI. The goal: v2 ships as flags and new writes, not painful migrations.

Dormant tables in v1

Create empty tables for v2 entities now. No v1 queries should write to them — document that in Claude Skills so agents do not “helpfully” populate them early.

Polymorphism from day one

If v1 only uses one participant type, still model type + id pairs so v2 does not require a new sessions table.

Feature flags on every v2 surface

Ship v2 UI dark behind flags.* in jsonb. Flip per tenant when ready — never big-bang migrate live users.

Nullable + defaulted columns

v2 fields added in v1 migrations should be nullable with safe defaults. v1 code paths ignore them.

Auth multi-role

user_type or equivalent from v1 even if only one role logs in today.

Studio · Agents

Agents + Claude Skills.

Where AI does real work — in-product, founder-facing, and in your repo. Encode conventions in Skills so every prototype inherits the same rails.

Only automate high-leverage loops

Enrichment, drafting, ranking — not “AI everywhere.” Each agent should map to a task you do weekly as a solo founder.

One Skill per boundary

Supabase conventions, design system, outbound voice, audit rules. Trigger Skills from file paths or task labels in Claude Code.

Cost caps on every model call

Per-endpoint daily limits, weekly per-user caps, Slack alert at 80%. Non-negotiable for solo founders.
